# ユースフル資料作成スキル

ユースフル株式会社の社外向け資料（提案書・登壇資料・研修資料・サービス紹介資料）に共通する
「構成の型」「デザインルール」「文体・トーン」を言語化したもの。
2026-07時点でBOX内の以下の実資料を分析して作成した v1。
- `20.法人事業/070_登壇資料テンプレ/40_経営層・マネジメント向け/【Copilot】経営層・シニアマネジメント向け生成AIビジネス活用講座（ユースフル）.pptx`
- `20.法人事業/004_サービス資料/001_サービス提案資料/【ユースフル ビジネス】 サービスご提案資料.pdf`

母数がまだ少ないため一般化の精度は粗い。新しい資料を見るたびに本ファイルを磨くこと（[[neo-watanabe]] の
「既存スキルと照らし合わせる」ループ経由でも更新されうる）。

## この資料作成の背骨にある思想

1. **感情的価値と実務的価値を両立させる。**
   ミッション文「明日の働き方が変わる感動体験をお届けする」を、研修資料でも提案書でも一貫して使う。
   単なる機能・実績の説明で終わらせず、「感動体験」「働き方が変わる」といった情緒的な言葉を必ず添える。
2. **信頼は具体的な数字と固有名詞で語る。抽象的な自賛はしない。**
   「累計視聴者1,000万人」「導入企業200社以上」「Microsoft MVP 5年連続受賞」のように、
   検証可能な実績を数字・年数・出典（自社調べ／期間）付きで提示する。
3. **課題→差別化を対比構造で明示してから、自社の強みを言う。**
   いきなり自社の強みを語らず、「一般的な◯◯は〜のみ」→「ユースフルは〜まで支援する」という
   Before/After・比較の型を必ず経由する。
4. **権威性と親しみやすさを両立させる。**
   講師・登壇者紹介は実績（登壇歴、受賞歴）だけでなく、出身地・趣味・家族構成なども見せる
   「About me」形式にする。冷たい実績羅列にしない。
5. **一方通行にしない。**
   研修・登壇資料では「〜教えてください！」といった参加者への問いかけスライドを挟み、
   双方向のコミュニケーションを意識する。
6. **結論・ゴールは一言に凝縮する。**
   まとめスライドは長文にせず、キャッチフレーズ1つに凝縮し、キーの単語だけ色を変えて強調する
   （例：「AIを**右腕**として育てる」）。
7. **相手（対象者）に合わせて出し分ける。**
   同じ内容でも「経営層向け」「大学・教育機関向け」など対象者別にテンプレートを分けるのが前提。
   誰向けかを最初に確認してから作成する。

## 構成の基本パターン

社外向け資料（提案書・登壇資料）はおおむね以下の順で構成される。全部を毎回使う必要はないが、
迷ったらこの順を土台にする。

1. **表紙**：グラデーション背景＋一文キャッチコピー（実務的価値を一言で）＋資料タイトル＋ロゴ
2. **課題提起**：ダーク背景に切り替え、「一般的な◯◯は〜のみ」という一般論を示す
3. **差別化提示**：フロー図・矢印で「教える→変える」のような対比を可視化し、自社の支援範囲を示す
4. **チャプター区切り「会社概要 / About us」**：大きな連番（01, 02…）＋英語ラベル＋日本語タイトル
5. **会社概要 三点セット**：①ミッション一文 ②Company Profile表（会社名・設立日・資本金など）
   ③沿革タイムライン（年ごとの実績を箇条書き）
6. **数字で見る実績**：KPIカード／ゲージチャートで実績を並列表示（数字を大きく、単位は小さく）
7. **取引先・導入実績**：企業ロゴを並べて社会的証明を提示
8. **講師・担当者紹介（About me）**：専門性＋人柄（出身・趣味・家族など）
9. **（研修・登壇資料のみ）本日のゴール**：キャッチフレーズ1文＋キーワード強調
10. **（研修・登壇資料のみ）WORK／問いかけ**：参加者への質問で双方向性を作る
11. **本編コンテンツ**
12. **まとめ**：ゴールの再掲・一言キャッチフレーズ

## デザインルール

- **配色**：ブルー→グリーン（ティール）のグラデーションを基調カラーとする。表紙・チャプター区切り・
  強調枠・KPIの数字色に一貫して使う。
- **ロゴ**：ユースフルのYマークロゴを常に右上に固定配置。
- **見出しフォーマット**：左上に英語の小ラベル（`ABOUT US` `GOAL` `WORK` など）＋直下に大きな日本語
  タイトル。見出し上部にブランドカラーの細い罫線を入れる。
- **チャプター区切りページ**：大きな連番（`01` `02`…）＋英語ラベル＋日本語タイトルを、グラデーション
  背景いっぱいに配置する専用レイアウト。
- **数字の見せ方**：実績数値は大フォント・太字で目立たせ、単位（万人・社・%など）は小さく添える。
  可能ならゲージチャート／ドーナツチャートで視覚化する。出典（自社調べ・集計期間）を必ず小さく明記。
- **対比・フロー図**：課題と自社の強みを比較するときは、矢印付きの工程フロー図で「Before/After」を
  視覚化する。文章だけで比較を語らない。
- **強調**：キャッチフレーズ中の1〜2語だけ色（赤系など）を変えて強調する。全体を色文字にしない。
- **フッター**：`©ユースフル株式会社 All Rights Reserved.` とページ番号を毎ページ統一フォーマットで
  配置する。

## 文体・トーン

- 基本は敬体（です・ます調）。
- キャッチコピー・見出しは体言止め・短文で言い切る（説明的な長文タイトルにしない）。
- 専門用語を多用せず、平易な言葉＋具体的な数字で語る。
- 語彙はポジティブ・前向き（「成果を届ける」「感動体験」「変える」「伴走する」など）。
- 誇張表現（「業界No.1」等、根拠のない最上級表現）は避け、根拠のある数字に置き換える。

## 作成・レビュー時のチェックリスト

- [ ] 誰向けの資料か（経営層／実務担当／学生など）を確認し、テンプレートを出し分けたか
- [ ] 冒頭にミッション文または感情的価値を一言添えたか
- [ ] 自社の強みを言う前に、課題・一般論との対比を挟んだか
- [ ] 実績は数字＋出典で語っているか（根拠のない自賛表現になっていないか）
- [ ] 講師・担当者紹介に人柄要素が入っているか
- [ ] まとめが長文でなく、一言のキャッチフレーズに凝縮されているか
- [ ] 配色・ロゴ位置・見出しフォーマット・フッターが型から外れていないか
- [ ] （研修・登壇資料）参加者への問いかけスライドが最低1枚あるか
